## API Endpoints

### ↗️ **GET */user/***

Retrieve a user and their associated Google accounts.

**Parameters:**

* `username` *(query, string, required)* → The local username.

**Response:**

```json
{
  "accounts": [
    {
      "account_email": "user@gmail.com",
      "account_info": "personal",
      "details": "Token refreshed successfully"
    }
  ]
}
```

***

### ↗️ **POST */user/***

Create a new user.

**Parameters:**

* `username` *(query, string, required)* → The username to create.

**Response:**

```json
{ "username": "name" }
```

If the user already exists:

```json
{ "error": "User already exists" }
```

***

### ↗️ **POST */user/associate\_account***

Associate or refresh a Google account with a user.

**Parameters:**

* `username` *(query, string, required)*
* `refresh_token` *(query, string, required)* → OAuth refresh token.
* `account_info` *(query, string, required)* → A label to identify the account (e.g., `work`, `personal`).

**Response:**

```json
{
  "accounts": [
    {
      "account_email": "user@gmail.com",
      "account_info": "work",
      "details": "Token refreshed successfully"
    }
  ]
}
```

***

### ↗️ **POST */user/dissociate\_account***

Remove a Google account from a user.

**Parameters:**

* `username` *(query, string, required)*
* `account_email` *(query, string, required)*

**Response:** Return the remain google accounts. If the user or account doesn’t exist:

```json
{ "error": "User not found" } or { "error": "Google account not found" }
```

***

### ↗️ **POST */chat***

Streams agent responses and progress updates via **Server-Sent Events (SSE)**.

**Parameters (JSON body):**

* `username` *(string, required)* → The local user initiating the chat.
* `conversation_id` *(string, required)* → Identifier for the conversation (used for context continuity).
* `message` *(string, required)* → The user’s message.

The response is streamed as **SSE events**, meaning the client receives updates in real-time as the agent processes the request.

There are **three event types** emitted:

1. **`progress`**

   * Describes what stage the agent is in (e.g., planning, calling a tool, synthesizing response).
   * Useful for providing **live feedback** to end-users about what the agent is doing.

   Example:

   ```json
   { "type": "progress", "data": "🔧 Using **CreateEventTool**..." }
   ```

2. **`content`**

   * The actual text generated by the model.
   * May arrive in **multiple chunks** and should be concatenated to form the final assistant message.

   Example:

   ```json
   { "type": "content", "data": "Hello" }
   { "type": "content", "data": "from" }
   { "type": "content", "data": "the model." }
   ```

3. **`error`**

   * Sent if the orchestrator fails (either immediately or mid-process).
   * Allows clients to gracefully handle and display errors.

   Example:

   ```json
   { "type": "error", "data": "An error occurred while processing your request." }
   ```

#### Example of Streaming

A successful chat request may emit events like:

```json
{ "type": "progress", "data": "Analyzing request..." }
{ "type": "progress", "data": "🔧 Using **GmailListThreads**..." }
{ "type": "content", "data": "Hello" }
{ "type": "content", "data": " from the model." }
```

The client should **render `progress` events immediately** to show what the agent is doing, and **concatenate `content` events** to build the final assistant reply:

```
"Hello from the model."
```

If something goes wrong, the last event will always be:

```json
{ "type": "error", "data": "An error occurred while processing your request." }
```

### ↗️ **GET */chat/history***

Return the reconstructed message history for a `username` + `conversation_id`.

**Query Params**

* `username` *(string, required)*
* `conversation_id` *(string, required)*

**200 Response**
An ordered list of chat turns in `{role, content}` format. Roles are `"user"` or `"assistant"`.

```json
[
  { "role": "user", "content": "Schedule a meeting for tomorrow at 10am." },
  { "role": "assistant", "content": "The meeting has been ..." },
 ]
```

* If the user has **no associated accounts**, the endpoint returns an empty array `[]`.

* History is built from the orchestrator’s state (`thread_id` computed from `username` + `conversation_id`) by reading the `messages` list.

* This endpoint is **read-only** (no side effects) and returns **200** even for empty results.
